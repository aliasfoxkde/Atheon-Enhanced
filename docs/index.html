Why Leakr

Built for Java teams.

Leakr does one thing well: scan for leaked secrets. No configuration required, no native binary, no subprocess.

☕

One JAR, any platform

Runs on Windows, Linux, and macOS. If you have a JRE, you have Leakr. No architecture-specific binaries.

📦

Embeddable library

Call runner.scanString() directly from your application. No subprocess, no shell. Just a Maven dependency.

⚡

Parallel by default

Directory scans use a thread pool sized to your host CPU automatically. Fast on large codebases.

🔌

Drop-in extensible

Add a scanner by dropping one class into leakr.scanners. Auto-discovered at runtime. No config files.

📋

CI-ready

Exit code 0 means clean. Exit code 1 means findings. Pipe-friendly with --json output.

🍎

Scans everything

Files, directories, environment variables, and stdin. Pipe git diff straight into it.

Installation

Up in 30 seconds.

Download the JAR and run it. No package manager, no build step, no PATH changes required.

# Download
curl -L https://github.com/HoraDomu/Leakr/releases/download/v1.0.0/leakr.jar -o leakr.jar

# Run
java -jar leakr.jar scan .
git clone https://github.com/HoraDomu/Leakr.git
cd Leakr
mvn package -q
java -jar target/leakr.jar scan .
Usage

CLI & library.

Use it as a command-line tool or embed it directly in your Java application.

# Scan a directory
java -jar leakr.jar scan /path/to/project

# Scan environment variables
java -jar leakr.jar scan --env

# Pipe git diff through Leakr
git diff | java -jar leakr.jar scan --stdin

# JSON output for CI parsing
java -jar leakr.jar scan . --json

# Exclude directories
java -jar leakr.jar scan . --exclude target,dist,node_modules

# Limit to specific extensions
java -jar leakr.jar scan . --ext .env,.yaml,.json,.tf

# List all registered scanners
java -jar leakr.jar list
import leakr.core.*;
import java.nio.file.*;
import java.util.List;

Registry registry = new Registry();
Runner   runner   = new Runner(registry);

// Scan a string
List<Finding> findings = runner.scanString("AKIAIOSFODNN7EXAMPLE");

// Scan a file
List<Finding> findings = runner.scanFile(Path.of("config.yaml"));

// Scan a directory (parallel)
List<Finding> findings = runner.scanDir(Path.of("/path/to/repo"));

// Scan environment variables
List<Finding> findings = runner.scanEnv();

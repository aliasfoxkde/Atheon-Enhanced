Atheon

One tool. All patterns. Any input.

The mission

Atheon exists to be a platform — not a product. A community-driven pattern matching engine where anyone can contribute a pattern that every user immediately benefits from.

The engine is finished. What grows from here are the patterns: rules for any domain, written by anyone, covering everything text can contain. Security. Finance. Healthcare. Compliance. Legal. Operations. Gaming. If you can describe what you're looking for in a way that returns true or false, Atheon can run it.

No single team can cover every domain. But a community can.

What it is

Atheon is a community-driven pattern matching engine. You define the rules. You point it at anything — a file, a directory, environment variables, or any stream of text. It finds every match and tells you exactly where.

Three ways people use it:

Pre-commit hook. Drop Atheon into a git hook and it blocks bad commits before they leave the machine. Exit code 1 on any finding — that's all a hook needs to abort.

MCP server. atheon-mcp speaks the Model Context Protocol. Wire it into Claude, Cursor, or Windsurf and the AI can scan code mid-generation, before it ever reaches a file.

CI pipeline. Run atheon --categories=secrets . in any pipeline. Native binary, no runtime, no dependencies, runs in milliseconds regardless of how many patterns are loaded.

Patterns live in plain YAML files — one file per rule, no Go required. Organize them into categories and scan only what you need:

atheon --categories=secrets .
atheon --categories=secrets,pii .
atheon --all .

Directory scans automatically respect .gitignore and .atheonignore. Native binaries ship for Windows, macOS (Intel + Apple Silicon), and Linux with no runtime or dependencies.

The scenario

A developer wraps up a sprint and pushes a configuration file. Buried in a comment from a debugging session three weeks ago is a production API key. The commit goes through. The pipeline passes. Two months later, someone notices unusual billing activity.

Atheon, wired into a pre-push hook:

$ atheon ./

openai-api-key  config/app.yaml:47
  # de****a8f3c

1 finding(s)
scanned 23 file(s)  41.3 KB  4ms

Exit code 1. The push never happens. The key never leaves the machine. The incident never occurs.

That's it. That's Atheon.

Get Atheon

New versions ship on the 10th and 21st of every month.

Install with your package manager:

# macOS / Linux
brew tap HoraDomu/atheon
brew install atheon
# Windows
scoop bucket add atheon https://github.com/HoraDomu/scoop-atheon
scoop install atheon

Or download the binary directly — no install, no runtime, no dependencies.

Contribute

Patterns are the heart of Atheon. Every pattern is one YAML file — no Go required, no engine changes, fast to review, and immediately useful to every user once merged. If there's a domain you work in, a format you know, or a rule that would have saved your team time — that belongs here.

# community/secrets/my-service.yaml
name: my-service-api-key
match: '\bmsvc_[A-Za-z0-9]{32}\b'

The folder name is the category. Run go run ./bundler to rebuild the bundle, then open a PR.
